Form Validation

Native browser tooltips are off everywhere. Just write standard HTML constraints (required, pattern, min, type=email) and the validator handles the rest. Custom messages via data-msg-*. Opt out with data-no-validate.

<input type="email" required data-msg-required="Email is required." data-msg-email="That doesn't look right." /> <input type="number" min="1" max="100" data-msg-min="Must be ≥ 1." data-msg-max="Must be ≤ 100." /> <!-- Opt out for an entire form --> <form data-no-validate>…</form>

Custom Select

Native <select> wrapped automatically: keyboard nav (↑↓ Enter Esc), theme-aware menu, search-friendly DOM (the real <select> stays in the form for submit + validation). Opt out with data-native-select on the element or any ancestor.

<!-- Auto-enhanced --> <select required data-msg-required="Pick one."> <option value="">— choose —</option> <option value="a">Option A</option> </select> <!-- Opt out (keeps native chrome) --> <select data-native-select>…</select>

Character Counter

Any <textarea> or <input maxlength> gets a live counter. It turns amber at 90% and red when full.

<input type="text" maxlength="50" /> <textarea maxlength="200" rows="3"></textarea> <!-- Counter is injected automatically. Hide it with `data-no-validate` -->

Shake + Toast on Invalid Submit

Submitting an invalid form shakes the form, scrolls to the first bad field, focuses it, and shows a toast with the error count. Try clicking Submit below with empty fields.

<!-- All automatic — just write standard HTML constraints. --> <form> <input required /> <button type="submit">Submit</button> </form>
